# P1 4.2 캐릭터 피드백 구현 보고

## 목표

손님과 직원이 왜 멈췄는지, 왜 불만인지, 왜 실패했는지를 플레이어가 캐릭터 단위로 확인할 수 있게 만드는 것이 목표다.

피드백은 두 층으로 나눴다.

```text
머리 위 풍선 = 현재 감정/상태를 빠르게 읽는 신호
캐릭터 클릭 로그 = 최근 행동 실패/불만 원인을 확인하는 기록
```

## 구현 내용

### 로그 태그화

`Character.AddLog`가 긴 문장을 그대로 쌓지 않고 짧은 원인 태그로 정규화한다.

예시:

```text
쇼핑 실패: 목적지 없음 -> 길 막힘
보충 실패: 창고 재고 부족 -> 재고 부족
휴식: 피로 보호 -> 피로
시설 파손 -> 시설 파손
수용 인원 초과 -> 혼잡함
```

같은 원인이 연속으로 반복되면 새 줄을 계속 쌓지 않고 누적 표시한다.

```text
길 막힘
길 막힘
-> 길 막힘 x2
```

이를 위해 `CharacterLogEntry`와 `CharacterLogUtility`를 추가했다.

### 머리 위 피드백 풍선

`CharacterFeedbackBubble`을 추가했다.

캐릭터가 초기화될 때 자동으로 붙기 때문에 캐릭터 프리팹을 일일이 수정하지 않아도 된다.

초기 상태:

- 기쁨: `:)`
- 불만: `:/`
- 당황: `?`
- 분노: `!`
- 피로: `Zz`

실제 이모지 대신 ASCII 심볼을 사용했다. 이유는 TMP 기본 폰트가 이모지/일부 글리프를 못 찾아 경고를 만들 수 있고, P1에서는 의미 전달이 우선이기 때문이다.

상태 판정:

- 만족/완료/회복 로그 -> 기쁨
- 재고 부족/길 막힘/돈 부족/혼잡/시설 파손 -> 당황
- 위험/사망/사고 -> 분노
- 피로/비번/휴식 -> 피로
- 기분이 낮으면 불만 또는 분노를 지속 표시
- 피로가 낮으면 피로를 지속 표시

### 캐릭터 클릭 로그 UI

`CharacterSummeryInfo`에 최근 로그 표시를 추가했다.

기존 스탯 슬라이더는 유지한다.

로그 UI는 씬에 `TMP_Text`가 연결되어 있지 않아도 런타임에서 생성한다. 나중에 UI 프리팹을 정리할 때 별도 Text 영역으로 고정하면 된다.

표시 형태:

```text
최근 기록
- 재고 부족
- 길 막힘 x2
- 피로
```

## 구현 로직

로그가 들어오면 다음 순서로 처리한다.

```text
AddLog 원문 수신
-> CharacterLogUtility.ToCauseTag
-> 태그별 누적 횟수 증가
-> 직전 로그와 같은 태그면 마지막 줄 갱신
-> CharacterLogEntry 이벤트 발행
-> 머리 위 풍선 갱신
-> 클릭 로그 UI 갱신
```

풍선은 일시 표시와 지속 표시를 나눴다.

```text
로그 기반 상태 = 2.5초 정도 표시
스탯 기반 상태 = 피로/기분이 낮으면 지속 표시
```

## 검증

Unity MCP로 캐릭터 피드백 시나리오와 전체 P1 회귀 시나리오를 실행했다.

결과:

```text
Facility=True
Customer=True
Character=True
Owner=True
WorkPriority=True
Command=True
StaffDuty=True
Operation=True
Feedback=True
```

검증한 내용:

- 긴 로그가 짧은 원인 태그로 정규화된다.
- 같은 원인이 반복되면 `x2`처럼 누적 표시된다.
- 풍선 상태가 로그 태그와 스탯 상태에 맞게 분류된다.
- 캐릭터 클릭 로그 UI가 최근 로그를 포맷한다.

콘솔 에러는 0개다.

남아 있는 경고는 기존 경고다.

- `UIBuildingInfo.cs` unreachable code
- Behavior Designer deprecated API 경고
