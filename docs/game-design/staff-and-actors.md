# 직원과 NPC 역할

## 플레이어 사장

플레이어 사장은 몬스터 종족 중 하나로 선택되는 핵심 캐릭터다.

사장의 기본 데이터는 [캐릭터 모델](character-model.md)의 사장 캐릭터 규칙을 따른다.

초반에는 직원이 부족하므로 사장이 직접 던전 운영에 참여한다.

가능한 역할:

- 상점/식당 근무
- 재고 보충
- 시설 수리
- 설계도 연구
- 침입 시 전투 참여
- 불만 캐릭터 진정/협상

사장도 자동 작업과 우선 지정을 사용한다.

다만 사장은 일반 직원과 달리 죽으면 런이 끝나는 코어다.

따라서 사장을 위험 지역에서 직접 굴리는 것은 초반 생존에는 도움이 되지만, 장기적으로는 큰 리스크가 된다.

## 직원

직원은 경제와 시설 운영을 지원한다.

직원은 림월드처럼 작업 우선순위를 가진다.

직원의 기본 데이터는 [캐릭터 모델](character-model.md)을 따른다. 즉 직원은 능력, 개인 특성, 종족 특성을 가지고 있고, 작업 우선순위는 그 캐릭터를 어떤 일에 배치할지 정하는 운영 도구다.

직원은 기본적으로 만족한 몬스터 손님이 단골이 된 뒤 영입되는 구조다. 별도의 무작위 직원 고용 풀은 핵심 루프로 두지 않는다.

영입된 직원은 근무 시간이 아닐 때 던전 안에서 생활한다. 비번 상태의 직원은 손님처럼 시설을 이용하지만 일반 손님처럼 매출을 만들지는 않는다.

플레이어는 직원마다 어떤 일을 먼저 할지 정할 수 있어야 한다. 이 시스템은 직원 AI를 완전히 직접 조작하는 것이 아니라, 직원이 스스로 행동하되 플레이어가 성향과 역할을 잡아주는 장치다.

가능한 목표:

- 배정된 상점에서 근무
- 휴식
- 재고 보충
- 수리
- 연구
- 긴급 작업 대응

작업 우선순위 예시:

```text
1순위: 반드시 먼저 처리
2순위: 가능하면 우선 처리
3순위: 여유가 있을 때 처리
꺼짐: 이 작업은 하지 않음
```

작업 카테고리:

- 근무: 상점, 편의 시설, 생산 시설 운영
- 보충: 재고 운반, 창고에서 시설로 물품 이동
- 수리: 파손 시설 복구, 침입/반란 후 피해 복구
- 청소: 슬라임 오염, 더러움, 사고 흔적 정리
- 연구: 연구시설에서 설계도 분석, 조합식 해금 진행
- 경비: 지정 구역 순찰, 침입 시 전투 합류
- 구조/응급: 쓰러진 몬스터 회수, 위험 상태 지원
- 휴식: 피로가 높을 때 회복

운영 원칙:

- 직원은 자신의 우선순위가 높은 작업부터 찾는다.
- 같은 우선순위라면 거리, 긴급도, 개인 특성, 종족 적성을 고려한다.
- 침입, 화재, 난동 같은 긴급 상황은 일반 작업보다 높은 긴급도를 가진다.
- 휴식은 완전히 무시되면 직원 효율이 떨어지므로, 최소 피로 한계는 AI가 보호한다.
- 개인 특성은 작업 선호와 효율에 영향을 줄 수 있다.

### 자동 작업과 우선 지정

모든 작업은 기본적으로 자동으로 진행된다.

플레이어는 직원에게 매 순간 이동 명령을 내리는 것이 아니라, 캐릭터를 선택한 뒤 대상 시설, 구역, 몬스터, 작업 대상을 우선 지정할 수 있다.

입력 예시:

```text
캐릭터 선택
-> 파손 시설 우클릭
-> 해당 캐릭터의 수리 우선 지정
```

우선 지정은 직접 조작이 아니다.

플레이어는 `이 일을 먼저 해라`라는 목표를 지정하고, 실제 이동, 경로 탐색, 작업 수행은 AI가 처리한다.

적용 대상:

- 파손 시설 수리
- 재고 보충
- 시설 근무
- 연구시설에서 설계도 분석
- 오염 청소
- 쓰러진 몬스터 구조
- 반란 몬스터 제압
- 구역 경비
- 협상/진정 시도

운영 원칙:

- 우선 지정은 해당 캐릭터의 작업 우선순위를 임시로 끌어올린다.
- 도달할 수 없거나 수행할 수 없는 작업이면 실패 로그를 남기고 재계획한다.
- 피로, 부상, 공포, 매우 낮은 만족도 같은 상태가 심각하면 우선 지정을 거부하거나 중단할 수 있다.
- 긴급 상황은 일반 우선 지정보다 높은 우선도를 가질 수 있다.
- 우선 지정은 플레이어가 세밀하게 조작하는 액션 게임 입력이 아니라, 자동 AI에 의도를 전달하는 장치다.

### 근무와 비번

직원은 항상 작업만 하는 존재가 아니다.

근무 중에는 작업 우선순위에 따라 행동하고, 비번이 되면 생활 욕구를 해결한다.

근무와 비번은 플레이어가 시간표로 직접 지정하지 않는다.

AI가 피로, 만족도, 욕구, 작업 긴급도를 보고 자동으로 전환한다.

```text
근무
-> 피로 누적
-> 비번
-> 식사/휴식/선호 시설 이용
-> 피로와 만족도 회복
-> 다시 근무
```

비번 행동 예시:

- 식당 이용
- 휴식 시설 이용
- 종족 선호 시설 방문
- 훈련장 이용
- 치료 또는 회복
- 가까운 구역에서 대기

운영 원칙:

- 직원/방어 인력은 기본적으로 월급을 받지 않는다.
- 직원 유지비는 돈 지출이 아니라 시설 점유, 재고 소비, 식사 소비, 피로/만족도 관리로 표현한다.
- 비번 직원은 시설 수용량과 동선을 차지한다.
- 비번 직원은 매출을 만들지 않는다.
- 비번 직원은 재고나 식사를 소비할 수 있다.
- 직원 만족도가 낮으면 작업 효율이 떨어진다.
- 직원 만족도가 낮은 상태로 오래 방치되면 사고, 결근, 이탈 위험이 생길 수 있다.
- 플레이어는 시간표가 아니라 작업 우선순위, 시설 배치, 휴식 시설 수, 구역 설계로 비번 품질을 조절한다.
- 침입이나 긴급 상황이 발생하면 경비 우선순위와 상태에 따라 비번 직원도 대응에 합류할 수 있다.

초기 구현에서는 깊은 생활 시뮬레이션보다 피로와 간단한 만족도만 관리한다.

### 직원 불만과 내부 위협

직원 만족도가 낮은 상태로 오래 방치되면 내부 위협으로 발전한다.

직원 불만은 다음 순서로 악화된다.

```text
효율 저하
-> 태업/결근/사고
-> 이탈
-> 반란/내부 전투
```

각 단계는 플레이어가 알아차릴 수 있어야 한다.

표시 방식:

- 캐릭터 이모티콘 풍선: 불만, 분노, 피로
- 캐릭터 클릭 로그: 식사 부족, 휴식 부족, 선호 시설 없음, 과로, 혼잡
- 오른쪽/우하단 이벤트 알림: 결근, 사고, 이탈, 반란 발생
- 운영일 정산: 직원 불만 사건 요약

단계별 예시:

```text
효율 저하:
작업 속도 감소, 근무 중 멈칫거림, 작업 실패

태업/결근/사고:
근무 거부, 작업 우선순위 무시, 시설 파손, 동료와 충돌

이탈:
직원이 던전을 떠나며 영구 손실 발생

반란/내부 전투:
직원이 적대화되거나 같은 편 몬스터와 전투 발생
```

반란은 외부 모험가 침입과 다른 내부 이벤트다.

반란은 바로 발생하지 않고, 낮은 만족도와 반복 경고가 누적된 뒤에만 발생한다.

반란이 발생해도 처음부터 사장방/플레이어 캐릭터를 직접 노리지는 않는다.

반란은 주변 시설 파손, 동료 공격, 탈출 시도에서 시작하고, 제압하지 못하면 사장방 위협으로 확산된다.

```text
주변 시설 파손
-> 가까운 동료 공격 또는 충돌
-> 탈출 시도
-> 장기 방치
-> 사장방/플레이어 캐릭터 위협
```

이 방식은 반란을 외부 침입의 복사본이 아니라, 던전 내부 운영 실패에서 발생한 국지 위협으로 만든다.

반란 대응 방식:

```text
자동 제압:
경비/방어 몬스터가 자동으로 반란 몬스터를 추적하고 제압한다.

제압 명령:
플레이어가 반란 몬스터나 구역을 지정해 대응 우선순위를 올린다.

격리:
문, 잠금 구역, 병목, 방 배치로 반란 몬스터를 가두거나 피해 범위를 줄인다.

협상/진정:
반란 직전 또는 초기 단계에서 비전투 방식으로 확산을 막는다.
```

제압 명령은 직접 전투 조작이 아니다.

플레이어는 목표와 우선순위를 지정하고, 이동과 전투는 AI가 자동으로 처리한다.

협상/진정은 완전 반란 이후 재영입 수단이 아니라, 반란이 영구 손실로 확정되기 전의 초동 대응이다.

이탈한 직원과 반란을 일으킨 직원은 기본적으로 재영입하지 못한다.

영구 손실은 플레이어가 납득할 수 있도록 다음 경고를 거쳐야 한다.

- 같은 불만 원인이 캐릭터 로그에 반복 기록된다.
- 불만/분노 이모티콘이 반복 노출된다.
- 효율 저하, 태업, 결근 같은 선행 사건이 발생한다.
- 오른쪽/우하단 이벤트 알림으로 위험 상태를 알려준다.
- 운영일 정산에서 직원 불만 사건을 요약한다.

### 작업 능력과 종족 보정

작업 속도는 림월드처럼 개체별 능력에 따라 달라진다.

기본은 개인 능력이고, 종족 성향은 보정으로 적용한다.

```text
최종 작업 속도 = 개인 작업 능력 + 개인 특성 보정 + 종족 보정 + 시설/환경 보정
```

예시:

```text
뱀파이어
- 연구/마력 분석 보정 높음
- 휴식 필요 낮음
- 청소/운반 보정 낮음

오크
- 경비/수리/훈련 보정 높음
- 연구 보정 낮음

슬라임
- 청소/오염 처리 보정 높음
- 경비 보정 낮음
```

개인 특성 예시:

```text
연구가: 연구 속도 증가
호기심 많음: 연구와 새 시설 조사 선호 증가
싸움꾼: 경비/훈련 선호 증가, 연구 선호 감소
깔끔함: 청소 선호와 청소 속도 증가
성급함: 긴 작업의 효율 감소 가능
```

이 구조에서는 같은 뱀파이어라도 연구가 특성이 있으면 뛰어난 연구원이 되고, 산만한 특성이 있으면 연구 속도가 낮아질 수 있다.

예시:

```text
오크 직원
- 경비 1
- 근무 2
- 수리 3
- 연구 꺼짐
- 청소 꺼짐

슬라임 직원
- 청소 1
- 보충 2
- 근무 3
- 연구 꺼짐
- 경비 꺼짐

뱀파이어 직원
- 연구 1
- 근무 1
- 경비 2
- 휴식 낮음
- 청소 3
```

이 구조를 사용하면 같은 몬스터라도 손님, 직원, 방어 인력으로 역할을 바꿨을 때 다른 운영 의미를 가질 수 있다.

가능한 실패 상황:

- 배정된 상점이 사라짐
- 길 막힘
- 스케줄 변경
- 시설 이용 불가
- 더 높은 우선순위 작업이 계속 생겨 낮은 우선순위 작업이 밀림
- 연구 우선순위가 낮아 설계도 분석이 지연됨
- 피로 누적으로 작업 효율 저하
- 비번에 필요한 시설이 부족해 만족도 회복 실패
- 직원 불만 누적으로 태업, 결근, 사고, 이탈, 반란 발생

## 몬스터

몬스터는 손님, 직원, 방어 인력으로 나뉠 수 있다. 손님 몬스터와 방어 몬스터는 같은 종족/개체 풀을 공유하되, 현재 역할과 상태에 따라 행동 목적이 달라진다.

가능한 목표:

- 배정된 방 경비
- 순찰
- 침입자 공격
- 후퇴 또는 회복
- 함정이나 방어 방 보조

## 침입자

침입자는 던전의 보물과 자원을 약탈하러 오는 모험가다.

핵심 목표:

- 사장방 도달
- 플레이어 캐릭터 처치

보조 목표:

- 시설 파손
- 자원 약탈
- 몬스터 처치
- 주요 시설 무력화
- 목표 달성 후 탈출

설계 방향:

- 침입자도 기본적으로 손님과 같은 그리드/복도/계단 이동 로직을 사용한다.
- 다만 침입자가 너무 오래 헤매면 이벤트 템포가 늘어지므로, 시간이 지날수록 사장방을 더 잘 찾게 해야 한다.
- 초반에는 탐색/헤맴이 있고, 시간이 지날수록 목표 방향 보정이 강해진다.
- 모든 침입자가 같은 행동을 하면 단조로우므로, 침입자 유형별로 주목표와 보조 목표를 다르게 둔다.

예상 이동 보정:

```text
침입 직후: 일반 탐색에 가까움
시간 경과: 사장방과 가까운 경로를 더 선호
장시간 경과: 거의 최단 경로에 가깝게 이동
```

이 방식은 침입자가 던전 구조에 영향을 받으면서도, 침입 이벤트가 너무 지루해지는 것을 막는다.

침입자 목표 유형 예시:

- 첫 구현: 사장방 돌파형 침입자 1종류
- 확장 아이디어 - 약탈자: 자원/돈을 우선 노리고, 기회가 있으면 사장방으로 향한다.
- 확장 아이디어 - 파괴자: 상점, 생산 시설, 방어 시설을 우선 파손시킨다.
- 확장 아이디어 - 사냥꾼: 방어 몬스터나 플레이어 캐릭터와의 전투를 우선한다.
- 확장 아이디어 - 용사: 사장방 돌파와 플레이어 캐릭터 처치를 최우선으로 한다.

시설 파손과 몬스터 처치는 침입 이벤트에 좋은 목표다. 이유는 다음과 같다.

- 플레이어가 동선뿐 아니라 시설 보호도 고려하게 된다.
- 방어 몬스터 배치가 의미를 가진다.
- 침입자가 사장방까지 못 가도 부분 피해를 남길 수 있다.
- 침입 이벤트 결과가 `막았다/뚫렸다` 이분법보다 풍부해진다.

초반부터 모든 목표를 동시에 쓰지 않는다. 첫 버전에서는 `사장방 도달`과 `간단한 시설 파손` 정도만 구현하고, 이후 몬스터 처치/자원 약탈/특수 침입자 유형을 확장한다.

구현 방향:

- 첫 구현은 침입자 타입 1개로 시작한다.
- 침입자 데이터에는 목표 가중치 필드를 둔다.
- 첫 타입은 사장방/플레이어 캐릭터 목표 가중치를 높게 둔다.
- 시설 파손, 몬스터 처치, 자원 약탈 가중치는 낮게 두거나 비활성화한다.
- 이후 타입 추가 시 데이터만 추가하고 이동/판단 로직은 최대한 재사용한다.
