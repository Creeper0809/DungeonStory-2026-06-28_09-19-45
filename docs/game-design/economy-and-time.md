# 경제와 시간 흐름

## 시간 구조

DungeonStory는 고정된 하루 턴 단위로 진행하지 않는다.

플레이어가 매일 정해진 행동 횟수를 쓰고 하루를 넘기는 방식이 아니라, 던전은 실시간 또는 준실시간으로 계속 운영된다.

다만 운영 흐름을 정리하고 보상을 체감시키기 위해 `운영일 정산 이벤트`는 존재한다.

핵심 방향:

```text
게임 진행 = 계속 흐름
정산 = 하루 단위 이벤트
```

즉, 하루는 플레이를 끊는 턴이 아니라, 수익과 사건을 요약해주는 리듬 장치다.

## 운영일 정산

운영일이 끝나면 화면 오른쪽 또는 우하단에 정산 알림 버튼이 뜬다.

플레이어가 버튼을 누르면 정산 창을 열어 자세한 내용을 확인한다.

정산 창은 플레이를 강제로 멈추는 필수 모달이 아니라, 이벤트 결과 알림의 한 종류로 둔다.

정산 창에 표시할 수 있는 항목:

- 총 매출
- 시설별 매출
- 방문 손님 수
- 종족별 방문 비율
- 평균 만족도
- 발생한 사고
- 침입 피해와 수리 비용
- 파손 시설 목록
- 직원 피로와 작업 요약
- 직원 불만 사건
- 재고 부족 시설
- 창고 재고 잔량
- 보충 실패 횟수
- 새로 해금된 도감 정보
- 상점 판매 목록 갱신

정산 창도 직접 처방을 하지 않는다.

좋은 예:

```text
재고 부족 시설: 저가 음식점
방문 비율 1위: 슬라임
사고 발생: 슬라임 오염 1회
직원 불만: 오크 직원 결근 1회
파손 시설: 저가 음식점
수리 비용: 35G
```

피해야 할 예:

```text
저가 음식점에 직원을 더 배치하세요.
휴식방에 습기 태그를 설정하세요.
방어 시설을 강화하세요.
```

## 경제 흐름

기본 경제 흐름:

```text
손님 방문
-> 시설 이용
-> 재고 소비
-> 매출 발생
-> 직원 작업/보충/수리
-> 파손 시설 복구
-> 운영 비용 차감
-> 운영일 정산
-> 상점 판매 목록 갱신
```

주요 자금 사용처:

- 방/복도/계단 건설
- 기본 시설 건설
- 일일 상점 상품 구매
- 재고 구매 또는 보충
- 시설 수리
- 직원 숙소/휴식/식사 기반 확장
- 연구/합성 비용

직원과 방어 인력은 기본적으로 월급을 받지 않는다.

직원 유지비는 돈이 직접 빠지는 방식이 아니라 던전 내부 운영 압박으로 표현한다.

```text
직원 수 증가
-> 식사/재고 소비 증가
-> 휴식 시설 점유 증가
-> 동선 혼잡 증가
-> 피로/만족도 관리 부담 증가
```

이렇게 하면 인력 증가는 단순한 고정비가 아니라, 던전 구조와 시설 용량을 요구하는 경영 문제로 작동한다.

초기 구현에서는 경제 변수를 너무 많이 두지 않는다.

초기 경제 변수:

- 돈
- 시설 가격
- 시설 매출
- 재고: 식재료, 잡화, 무기, 마력
- 파손 시설
- 수리 비용
- 직원 피로

후반 확장 변수:

- 시설 유지비
- 세금/상납금
- 평판
- 던전 가치
- 희귀 시설 가치

## 상점 리셋

일일 상점은 운영일 정산을 기준으로 판매 목록이 갱신된다.

상점 리셋은 고정 턴 종료가 아니라, 운영일 정산 이벤트에 묶인 보상/선택 리듬이다.

방향:

- 운영일 정산 후 판매 목록이 바뀐다.
- 플레이어는 정산 내용을 보고 다음에 살 시설이나 설계도를 고른다.
- 던전 등급, 평판, 연구 상태에 따라 등장 상품이 달라진다.
- 희귀 상품은 가끔 등장해 구매 결정을 만든다.
- 연구로 해금된 1성/일부 2성 시설은 랜덤 상품과 별도로 기본 구매 목록에 들어간다.
- 너무 운에만 의존하지 않도록 나중에는 상품 잠금, 재입고 요청, 상인 등급 상승 같은 추가 보정 장치를 고려한다.

## 침입과 시간

침입은 고정 날짜에 발생하지 않는다.

침입 이벤트는 내부 위협도 누적 방식으로 발생하며, 운영일 정산은 침입 타이밍을 직접 고정하지 않는다.

다만 정산 창은 침입 관련 누적 정보를 요약할 수 있다.

예시:

```text
침입 피해: 저가 음식점 파손
수리 비용: 35G
획득 정보: 성향 - 시설 파손 우선
최근 침입 이후 경과 시간: 2일
```

## 초기 구현 범위

첫 구현에서는 하루를 엄격한 턴으로 만들지 않는다.

초기 목표:

- 운영 시간이 흐른다.
- 일정 시간이 지나면 운영일 정산 알림이 뜬다.
- 정산 창에서 매출, 손님 수, 피해, 재고 부족, 창고 재고, 보충 실패, 직원 피로를 보여준다.
- 정산 후 일일 상점 판매 목록이 갱신된다.
- 정산 창은 직접 처방 없이 관측된 사실만 보여준다.
