# 시설 획득과 합성

## 시설 획득과 합성

시설 강화는 경영과 전투를 연결하는 핵심 성장 축이다.

경영 시설의 욕구, 직원 작업, 재고 연결은 [경영 시설 컨셉](management-facilities.md) 문서에서 별도로 관리한다.

시설 공격 컨셉은 [시설 공격 컨셉](facility-attack-concepts.md) 문서에서 별도로 관리한다.

방향성:

- 시설은 단순 구매만으로 끝나지 않는다.
- 플레이어는 시설을 획득하고, 배치하고, 여러 시설을 합성해 더 강한 시설로 성장시킨다.
- 합성 방식은 던전메이커처럼 시설물끼리 조합해 상위 시설이나 강화 시설을 만드는 구조를 따른다.
- 전투는 자동이므로, 플레이어의 실질적인 전투 개입은 좋은 시설을 얻고 합성하고 배치하는 데서 나온다.

### 시설 획득 루트

시설 획득은 이 게임의 핵심 보상 루프가 되어야 한다.

주 루프:

- 운영일 정산을 기준으로 갱신되는 상점에서 시설과 설계도를 구매한다.
- 정산 이후 판매 목록이 갱신된다.
- 플레이어는 현재 자금, 던전 상태, 합성 목표를 보고 무엇을 살지 선택한다.
- 상점 리셋은 운영일마다 작은 의사결정을 만든다.

가능한 획득 루트:

- 기본 건설: 초반 필수 시설은 돈으로 직접 건설한다.
- 일일 상점/거래: 몬스터 상인, 암시장, 방문 상인에게서 시설이나 설계도를 구매한다.
- 침입 방어 보상: 모험가 침입을 막으면 시설이나 설계도를 얻는다.
- 오펜스 원정 보상: 다른 던전이나 세력을 공격해 희귀 시설이나 설계도를 얻는다.
- 연구/해금: 연구시설에서 설계도를 분석하면 새로운 시설 제작법이나 합성 조합식이 열린다.
- 이벤트 보상: 특수 손님, 퀘스트, 선택 이벤트로 시설을 얻는다.

초기 구현 추천:

- 기본 시설은 건설 메뉴에서 돈으로 제공한다.
- 시설 획득의 주 루프는 운영일 정산 기준으로 리셋되는 상점 구매로 둔다.
- 침입 방어 보상과 이벤트 보상은 특수 시설이나 설계도를 주는 보조 루프로 둔다.
- 오펜스 원정 보상은 장기 확장으로 둔다.

### 시설 파손과 수리

침입, 반란, 난동으로 피해를 입은 시설은 기본적으로 사라지지 않는다.

시설은 `파손 상태`가 되고, 직원의 수리 작업을 통해 복구한다.

```text
정상 시설
-> 피해 누적
-> 파손 상태
-> 수리 작업
-> 정상 복구
```

기본 규칙:

- 피해를 입은 시설은 위치, 별 등급, 레벨, 합성 계보를 유지한다.
- 초기 구현에서는 시설 상태를 `정상`과 `파손` 두 가지로만 둔다.
- 초기 구현에서 파손 상태는 완전 비활성화로 처리한다.
- 상점/편의 시설이 파손되면 손님이 이용하지 못한다.
- 방어 시설이 파손되면 침입 중 발동하지 않는다.
- 지원 시설이 파손되면 직원 작업, 생산, 연구, 회복 보정을 제공하지 않는다.
- 파손 시설은 수리 전까지 합성 재료로 사용하지 못하게 둔다.
- 수리는 직원의 `수리` 작업으로 처리한다.
- 수리에는 시간과 수리 비용 또는 재고가 들어간다.
- 수리는 기본적으로 자동 작업이지만, 플레이어가 특정 파손 시설을 우선 수리 대상으로 지정할 수 있다.

내구도에 따라 성능이 점점 떨어지는 단계형 피해는 후반 확장으로 둔다.

확장 예시:

```text
정상
-> 경미한 손상
-> 심각한 손상
-> 파손
```

첫 구현에서는 이 단계를 만들지 않고, `정상/파손`만 사용한다.

완전 소실은 기본 피해 결과로 두지 않는다.

완전 소실은 후반 특수 이벤트, 매우 강한 침입, 특정 저주 같은 예외적인 상황에서만 고려한다.

이렇게 하면 합성으로 키운 시설이 한 번의 사고로 사라지는 억울함을 줄이면서도, 피해 복구와 수리 우선순위가 경영 문제로 남는다.

상점 리셋 설계 방향:

- 운영일 정산 후 판매 목록이 바뀐다.
- 던전 등급, 스킬트리, 평판이 높아질수록 더 좋은 시설이 등장한다.
- 일부 상품은 희귀하게 등장해 구매 결정을 만든다.
- 너무 운에만 의존하지 않도록 낮은 별 시설은 연구를 통해 기본 구매 목록에 편입된다.
- 나중에는 상품 잠금, 재입고 요청, 상인 등급 상승 같은 추가 보정 장치를 고려한다.

### 상점 운 보정과 기본 구매 해금

시설 합성이 핵심 성장 루프라면, 필요한 1성/2성 시설이 계속 나오지 않는 상황은 플레이어의 계획을 막을 수 있다.

따라서 운영일 상점은 랜덤성을 가지되, 연구가 진행될수록 낮은 별 시설은 기본 구매 목록으로 해금한다.

방향:

```text
초반: 상점 리셋과 기본 건설에 의존
연구 진행: 자주 쓰는 1성 시설 직접 구매 해금
중반: 핵심 2성 시설 일부 직접 구매 해금
후반: 3성 이상은 합성, 침입 보상, 원정 보상 중심 유지
```

기본 구매 해금 예시:

```text
상업 기초 연구: 1성 식당, 1성 잡화점 기본 구매 가능
방어 기초 연구: 1성 가시 함정, 1성 경비실 기본 구매 가능
지원 기초 연구: 1성 창고, 1성 훈련장 기본 구매 가능
상업 확장 연구: 일부 2성 경영 시설 기본 구매 가능
방어 확장 연구: 일부 2성 방어 시설 기본 구매 가능
```

기본 구매 목록에 들어간 시설은 운영일 상점의 랜덤 상품과 별도로 언제든 구매할 수 있다.

단, 높은 별 시설까지 직접 구매가 쉬워지면 합성 루프가 약해진다. 그래서 기본 구매 해금은 주로 1성, 일부 2성 시설까지만 허용한다.

역할:

- 필요한 기초 시설을 못 뽑아서 합성이 막히는 상황을 줄인다.
- 플레이어가 합성 목표를 세우고 필요한 하위 시설을 준비할 수 있게 한다.
- 상점 랜덤성은 희귀 시설, 높은 별 시설, 특수 설계도 쪽에 남긴다.

### 시설 합성 방식

합성은 던전메이커식 구조를 기본으로 한다.

이 게임에서 합성의 핵심은 `한 번 강화하고 끝`이 아니라, 시설물끼리 계속 합성해 상위 시설로 쭉쭉 올리는 성장감이다.

핵심 규칙:

- 합성은 같은 종류 누적 강화가 아니라 조합식 중심으로 진행한다.
- 정해진 시설 조합으로 상위 시설이나 새로운 기능의 시설로 진화시킨다.
- 합성 결과는 단순 수치 상승뿐 아니라 기능 변화가 있어야 한다.
- 합성에 사용한 시설은 사라진다.
- 합성 결과는 배치 가능한 새 시설이 되거나, 기존 시설을 상위 시설로 대체한다.
- 합성에 사용한 시설의 레벨은 합성 결과에 일부 계승된다.
- 핵심 시설군은 1성에서 5성까지 이어지는 분기형 합성 트리를 가진다.
- 하나의 시설은 함께 합성하는 다른 시설과 연구 상태에 따라 여러 상위 시설로 갈라질 수 있다.

합성 예시:

```text
1성 고기 식당 + 1성 훈련장 -> 2성 전투 식당
1성 고기 식당 + 1성 장식실 -> 2성 고급 고기 식당
1성 고기 식당 + 1성 묘지 -> 2성 피의 식당
1성 가시 함정 + 1성 독 웅덩이 -> 2성 맹독 함정
1성 경비실 + 1성 훈련장 -> 2성 병영

2성 전투 식당 + 2성 경비실 -> 3성 전장의 식당
2성 전투 식당 + 2성 전리품 창고 -> 3성 약탈자의 식당

2성 고급 고기 식당 + 2성 마력실 -> 3성 귀족의 식당
2성 고급 고기 식당 + 2성 대형 창고 -> 3성 대식 연회장
```

합성 설계 원칙:

- 초반부터 조합식 기반으로 설계하되, 초반 조합식은 단순하게 둔다.
- 중복 시설은 조합식에 사용할 수 있지만, 성장의 중심은 조합식이다.
- 후반에는 희귀 시설이나 원정 보상 시설을 요구해 장기 목표를 만든다.
- 모든 시설이 합성 대상일 필요는 없지만, 핵심 시설군은 성장 경로를 가져야 한다.
- 합성 결과는 사용한 시설의 레벨을 완전히 버리지 않고, 일정 비율로 계승한다.
- 상위 시설일수록 경영 효과와 방어 효과가 함께 커지거나 성격이 뚜렷해진다.
- 같은 출발 시설이라도 어떤 갈래를 선택했는지에 따라 고객층, 방어 방식, 필요한 시설 조합이 달라진다.
- 합성 트리는 시설 도감에서 확인하되, 미발견 조합식은 일부 요구 시설이나 결과를 가려둘 수 있다.

### 합성 발견 방식

합성 조합식은 모두 숨기지 않는다.

일반적인 조합식은 플레이어가 처음부터 확인할 수 있게 한다.

예시:

```text
가시 함정 + 독 웅덩이 -> 맹독 함정
경비실 + 훈련장 -> 병영
고기 식당 + 훈련장 -> 전투 식당
```

이런 기본 조합식은 던전 운영의 계획을 세우는 기반이므로 숨기면 안 된다.

반대로 특수 조합식은 설계도와 연구시설 분석을 통해 해금한다.

예시:

```text
특수 시설 조합식
희귀 시설 조합식
높은 별 시설 조합식
오펜스 원정 보상 시설이 필요한 조합식
특정 종족/세력과 연결된 조합식
```

방향:

```text
일반 조합식 = 공개
특수 조합식 = 설계도 획득 후 연구시설에서 분석
미발견 특수 조합식 = 도감에서 일부 힌트만 표시 가능
```

이 구조는 계획성과 발견 재미를 모두 살린다.

플레이어는 기본 조합식으로 던전을 안정화하고, 침입 방어/오펜스/이벤트에서 얻은 설계도를 분석해 새로운 빌드 방향을 연다.

### 분기형 연쇄 합성 트리

시설 종류가 무한히 늘어나지 않도록, 모든 시설을 같은 깊이로 만들지 않는다.

핵심 시설군만 1성에서 5성까지 이어지는 분기형 트리를 가진다.

시설군 예시:

- 식당 계열
- 상점 계열
- 휴식 계열
- 묘지/마력 계열
- 함정 계열
- 경비/병영 계열
- 지원/생산 계열
- 하이브리드 계열

각 트리는 다음 구조를 가진다.

```text
기본 시설
-> 특화 A
   -> A-1
   -> A-2
-> 특화 B
   -> B-1
   -> B-2
-> 특화 C
   -> C-1
   -> C-2
```

예시:

```text
고기 식당
-> 전투 식당
   -> 전장의 식당
   -> 약탈자의 식당
-> 고급 고기 식당
   -> 귀족의 식당
   -> 대식 연회장
-> 피의 식당
   -> 흡혈 식당
   -> 사령 식당
```

```text
휴식방
-> 점액 휴식실
   -> 점액 욕장
   -> 슬라임 왕의 탕
-> 산성 욕장
   -> 산성 수조
   -> 부식 연못
-> 치료 휴식실
   -> 재생 욕장
   -> 생명수 샘
```

```text
묘지
-> 공포 묘지
   -> 망자의 회랑
   -> 사령왕의 묘실
-> 영혼 묘지
   -> 영혼 제단
   -> 마력 납골당
-> 저주 묘지
   -> 저주 제단
   -> 파멸의 묘실
```

이 방식이면 플레이어는 던전메이커처럼 계속 합성하는 재미를 느끼면서도, 매번 같은 정답 계보만 따라가지 않는다.

분기 선택은 되도록 다음 차이를 만들어야 한다.

- 주 고객층이 달라진다.
- 침입 대응 방식이 달라진다.
- 요구 시설과 연구 방향이 달라진다.
- 경영형, 방어형, 지원형, 하이브리드형 중 역할 방향이 달라진다.
- 시설 외형 레이어와 분위기가 달라진다.
- 이후 합성 가능 갈래가 달라진다.

개발 범위는 트리의 깊이와 폭으로 조절한다. 모든 시설에 넓은 트리를 주기보다, 핵심 시설군에만 의미 있는 분기를 준다.

### 역할 분리와 합성 연결

기본 시설은 역할을 구분한다.

경영 시설과 방어 시설을 처음부터 모두 겸용으로 만들면 플레이어가 시설의 목적을 읽기 어려워진다.

따라서 기본 역할은 분리한다.

```text
경영 시설: 돈, 만족도, 재고, 손님 체류를 담당한다.
방어 시설: 침입자 지연, 피해, 경보, 경비 대응을 담당한다.
지원 시설: 직원, 생산, 회복, 보충, 연구를 돕는다.
하이브리드 시설: 합성을 통해 경영 효과와 방어 효과를 함께 가진다.
```

핵심 원칙:

```text
역할은 분리한다.
공간과 경제는 공유한다.
합성으로 서로 연결한다.
```

경영 시설은 침입 시 파손/약탈 대상이 될 수 있지만, 그 자체가 항상 방어 성능을 가져야 하는 것은 아니다.

방어 시설은 직접 수익을 거의 만들지 않지만, 침입 피해를 줄이고 경영 시설을 보호한다.

하이브리드 시설은 합성으로 만든 특수 시설이다. 경영 시설과 방어/지원 시설을 조합해 만들어지며, 두 역할을 동시에 가진다.

합성 방향 예시:

```text
경영 시설 + 경영 시설 -> 고급 경영 시설
방어 시설 + 방어 시설 -> 고급 방어 시설
경영 시설 + 방어 시설 -> 하이브리드 시설
경영 시설 + 지원 시설 -> 운영 특화 시설
방어 시설 + 지원 시설 -> 전술 방어 시설
경영 시설 + 종족 시설 -> 종족 특화 시설
```

시설 역할 예시:

```text
고기 식당
- 역할: 경영 시설
- 평상시: 오크 손님 만족도와 소비 증가
- 침입 시: 파손/약탈 대상

가시 함정
- 역할: 방어 시설
- 평상시: 수익 없음
- 침입 시: 침입자에게 피해

경비실
- 역할: 방어 시설
- 평상시: 경비 직원 대기/순찰 거점
- 침입 시: 근처 경비 직원 반응 속도 증가

훈련장
- 역할: 지원 시설
- 평상시: 직원/몬스터 훈련, 오크 선호
- 침입 시: 전투형 직원의 성능 보조
```

하이브리드 시설 예시:

```text
전투 식당 = 고기 식당 + 훈련장
- 역할: 하이브리드 시설
- 평상시: 오크 손님 만족도와 소비 증가
- 침입 시: 근처 오크 직원 공격력 증가

포획 구역 = 끈끈이 함정 + 경비실
- 역할: 전술 방어 시설
- 평상시: 경비 직원 대기 효율 증가, 직접 수익 없음
- 침입 시: 침입자 이동 속도 감소, 경비 직원 합류 속도 증가

점액 욕장 = 휴식방(습기) + 끈끈이 함정
- 역할: 하이브리드 시설
- 평상시: 슬라임 손님 만족도와 체류 시간 증가
- 침입 시: 방 안 침입자 이동 속도 감소

공포 묘지 = 묘지 + 경비실
- 역할: 하이브리드 시설
- 평상시: 뱀파이어 손님 만족도 증가
- 침입 시: 침입자 명중률 또는 사기 감소
```

이 구조는 경영과 방어를 따로 읽히게 만들면서도, 합성 단계에서는 두 시스템이 서로 섞이게 만든다.

플레이어는 현재 필요에 따라 다음 합성 방향을 선택할 수 있다.

```text
돈이 부족하다 -> 경영 시설 계열 합성
침입 피해가 크다 -> 방어 시설 계열 합성
특정 종족을 노린다 -> 종족 특화 계열 합성
둘 다 필요하다 -> 하이브리드 시설 합성
```

### 부착물 역할

부착물은 메인 성장 루프가 아니라 보조 커스터마이징으로 둔다.

부착물은 시설을 상위 시설로 대체하지 않고, 이미 배치된 시설에 작은 기능과 외형 레이어를 더한다.

예시:

```text
가시 함정: 침입자에게 피해
끈끈이 함정: 침입자 이동 속도 감소
경비 종: 근처 직원 반응 속도 증가
룬 장식: 특정 종족/속성 시너지 증가
```

정리하면 다음과 같다.

```text
분기형 합성 트리 = 던전메이커식 장기 성장의 중심
부착물 = 같은 시설을 다르게 꾸미고 보조 기능을 붙이는 장치
```

레벨 계승 예시:

```text
시설 A Lv.3 + 시설 B Lv.2 -> 결과 시설 Lv.2
합성에 사용한 시설 평균 레벨 또는 가장 높은 시설 레벨의 일부를 결과에 반영
```

레벨 계승이 필요한 이유:

- 플레이어가 기존 시설을 키운 시간이 낭비되지 않는다.
- 합성 타이밍을 고민하게 만든다.
- 낮은 등급 시설도 장기적으로 합성 가치가 생긴다.

### 시설 등급과 역할

시설은 역할과 등급을 분리해서 설계한다.

역할 예시:

- 상업 시설: 돈을 번다.
- 편의 시설: 몬스터 손님 만족도와 체류 시간을 조절한다.
- 생산 시설: 재고, 재료, 자원을 만든다.
- 방어 시설: 침입자를 방해하거나 피해를 준다.
- 지원 시설: 직원, 몬스터, 다른 시설의 효율을 올린다.

등급 표기:

```text
1성 -> 2성 -> 3성 -> 4성 -> 5성
```

시설 등급은 이름 등급이 아니라 별 개수로 표현한다.

최종 방향은 핵심 시설군마다 `1성 -> 5성`까지 이어지는 분기형 연쇄 합성이다.

다만 첫 프로토타입에서는 모든 시설군을 5성까지 만들기보다, 대표 시설군 1-2개를 `1성 -> 3성` 정도까지 구현하되 최소 2갈래 이상으로 갈라지게 만들어 합성 선택이 재미있는지 먼저 검증한다.

### 미정 사항

- 합성 결과를 플레이어가 미리 알 수 있는가, 발견형으로 둘 것인가?
- 배치 중인 시설을 합성에 쓸 수 있는가?

### 설계도와 연구시설

설계도는 합성에 소비되는 재료가 아니다.

설계도는 연구시설에서 분석해 새로운 시설 제작법, 기본 구매 항목, 합성 조합식을 해금하는 정보 아이템이다.

즉, 시설 합성은 시설물끼리 진행하고, 설계도는 `무엇을 만들 수 있는지`를 여는 역할을 한다.

설계도 흐름:

```text
상점/침입/이벤트/원정에서 설계도 획득
-> 연구시설에 등록
-> 연구 우선순위가 있는 직원이 연구 작업 진행
-> 시설 제작법/합성 조합식/기본 구매 항목 해금
```

설계도로 해금되는 것:

- 신규 기본 시설 건설
- 특정 시설군의 1성 기본 구매
- 일부 2성 시설 기본 구매
- 분기형 합성 조합식
- 희귀 시설 조합식
- 합성 계승 효율 또는 비용 감소 연구

예시:

```text
식당 설계도: 1성 식당 건설/구매 해금
방어 기초 설계도: 1성 가시 함정, 1성 경비실 구매 해금
전투 식당 설계도: 고기 식당 + 훈련장 -> 전투 식당 조합식 해금
귀족의 식당 설계도: 고급 고기 식당 + 마력실 -> 귀족의 식당 조합식 해금
```

설계도는 획득 즉시 효과가 발생하지 않는다. 연구시설에서 직원이 분석해야 해금된다.

연구 작업 원칙:

- 연구는 직원 작업 우선순위의 한 카테고리다.
- 연구 우선순위가 높은 직원이 연구시설로 이동해 설계도 분석을 진행한다.
- 연구시설에 접근할 수 없거나 연구 가능한 직원이 없으면 분석이 멈춘다.
- 연구 속도는 개인 연구 능력이 기본이고, 개인 특성과 종족 보정이 추가로 영향을 준다.
- 연구가 완료되면 오른쪽/우하단 알림 버튼으로 결과를 알려준다.

이 구조는 보상을 바로 소비재로 쓰는 것이 아니라, 장기적인 연구 목표로 연결한다.

### 시설 합성 연구

시설 합성은 처음부터 열려 있지 않다.

림월드 연구처럼 연구/스킬트리 해금을 통해 합성 기능과 조합식이 열린다.

연구는 두 단계를 가진다.

```text
합성 기능 해금: 합성 UI/작업대/기본 규칙이 열린다.
조합식 해금: 어떤 시설을 어떤 상위 시설로 만들 수 있는지가 열린다.
```

즉, 첫 연구로 모든 합성이 한 번에 열리는 것이 아니라, 플레이어가 현재 던전 문제에 맞춰 다음 연구 목표를 고르게 한다.

해금 흐름:

```text
첫 침입 경험
-> 더 강한 시설의 필요성을 느낌
-> 합성 연구 해금
-> 기본 합성 기능 사용 가능
-> 설계도를 연구시설에서 분석
-> 연구 진행으로 더 많은 조합식 해금
```

연구로 해금되는 요소:

- 기본 합성 기능
- 시설 역할군별 조합식
- 높은 별 등급 시설 조합식
- 레벨 계승 효율 증가
- 합성 비용 감소
- 희귀 시설 사용 조합식
- 합성 슬롯 또는 동시 합성 가능 수
- 낮은 별 시설의 기본 구매 해금

연구 선택 방향:

- 침입자가 너무 빨리 뚫고 들어오면 방어 시설 연구를 선택한다.
- 손님 만족도가 낮아 사고가 잦으면 편의 시설 연구를 선택한다.
- 수익이 부족하면 상업 시설 연구를 선택한다.
- 합성에 쓰는 시설 손실이 크면 계승 기술이나 비용 감소 연구를 선택한다.

이렇게 하면 연구는 단순한 시간 잠금이 아니라, 플레이어가 `지금 던전의 문제를 어떻게 해결할지` 고르는 경영 선택지가 된다.

연구 예시:

```text
시설 합성 I: 1성 시설 조합식 해금
시설 합성 II: 2성 시설 조합식 해금
방어 시설 연구: 방어 시설 계열 조합식 해금
상업 시설 연구: 상업 시설 계열 조합식 해금
상업 기초 연구: 일부 1성 경영 시설 기본 구매 해금
방어 기초 연구: 일부 1성 방어 시설 기본 구매 해금
계승 기술: 합성 시 사용 시설 레벨 계승 효율 증가
```

이 구조는 합성을 단순 메뉴 기능이 아니라 장기 성장 목표로 만든다.
