# 시설 공격 컨셉

시설 공격 컨셉은 던전메이커식 자동 전투의 핵심 재미를 만든다.

플레이어는 전투를 직접 조작하지 않으므로, 시설이 어떤 방식으로 침입자를 막는지 명확하게 읽혀야 한다.

## 목표

시설 공격 컨셉은 다음 역할을 가진다.

- 자동 전투 중 어떤 시설이 어떤 식으로 기여했는지 보여준다.
- 시설 합성의 갈래를 만든다.
- 침입자 도감의 약점/저항 정보와 연결된다.
- 몬스터 종족, 직원, 방어 시설 간 시너지를 만든다.
- 던전 배치 선택을 바꾼다.

좋은 공격 컨셉은 단순히 피해 숫자만 다르면 안 된다.

각 컨셉은 다음 중 하나 이상이 달라야 한다.

- 즉발 피해인가, 지속 피해인가?
- 빠르게 죽이는가, 오래 묶는가?
- 혼자 강한가, 다른 시설과 조합할 때 강한가?
- 긴 동선이 필요한가, 병목이 필요한가?
- 특정 종족/시설과 충돌하거나 시너지를 내는가?

## 초기 개수

초기 구현은 5개 공격 컨셉으로 시작한다.

```text
물리/가시
독/부식
화염/연소
번개/축전
냉기/속박
```

5개가 적당한 이유:

- 3개 이하는 합성 갈래가 단조롭다.
- 5개면 깡피해, 방어력 감소, 지속 피해, 연계/축전, 감속을 모두 표현할 수 있다.
- 8개 이상은 상태이상, 저항, 합성식, UI 설명 부담이 너무 커진다.
- 첫 프로토타입에서 각 컨셉을 1-2개 시설로 검증하기 좋다.

확장 후보는 2개로 둔다.

```text
공포/저주
흡혈/피
```

즉, 장기적으로는 `초기 5개 + 확장 2개` 정도가 적당하다.

## 초기 효과 원칙

초기 공격 컨셉은 너무 복잡하게 파생시키지 않는다.

모든 공격 컨셉은 기본 피해를 가진다.

그 위에 각 컨셉별로 하나의 핵심 차이만 둔다.

```text
물리/가시: 피해
독/부식: 피해 + 방어력 감소
화염/연소: 높은 피해 + 지속 피해
번개/축전: 피해 + 연계 + 축전
냉기/속박: 피해 + 감속
```

초기 5개 중 축전처럼 별도 누적 구조를 가지는 것은 번개만 둔다.

독, 화염, 냉기는 각각 방어력 감소, 지속 피해, 감속이라는 단순 효과로 시작한다.

## 초기 방어 시설 세트

첫 구현에서는 방어 시설을 많이 늘리지 않는다.

공격 컨셉 5개를 대표하는 1성 시설 5개와, 직원/몬스터 방어를 연결하는 경비실 1개로 시작한다.

```text
1성 가시 함정
1성 독 웅덩이
1성 화염 분사구
1성 번개 기둥
1성 냉기 분사구
1성 경비실
```

이 6개가 초기 방어의 최소 세트다.

초기 방어 시설은 복도 위 부착물이 아니라 모두 주 시설로 배치한다.

즉 방어 시설도 `한 방 = 주 시설 1개` 규칙을 따른다.

```text
방어 시설 설치
= Building 레이어에 배치
= 건설/파손/수리/합성 대상
= 침입자가 지나가며 발동하는 방어 방
```

복도 위에 별도 함정 레이어를 겹치는 방식은 사용하지 않는다.

| 시설 | 컨셉 | 기본 역할 | 배치 감각 |
| --- | --- | --- | --- |
| 가시 함정 | 물리/가시 | 즉발 피해 | 침입자가 반드시 지나가는 좁은 길 |
| 독 웅덩이 | 독/부식 | 피해 + 방어력 감소 | 다른 피해 시설 앞쪽 |
| 화염 분사구 | 화염/연소 | 높은 피해 + 지속 피해 | 침입자가 몰리는 방이나 병목 |
| 번개 기둥 | 번개/축전 | 피해 + 축전 | 여러 번 맞기 쉬운 긴 동선 |
| 냉기 분사구 | 냉기/속박 | 피해 + 감속 | 경비 몬스터가 합류하기 전 구간 |
| 경비실 | 경비/직원 | 직원 대기, 침입 시 교전 합류 | 사장방 전 병목이나 주요 시설 근처 |

경비실은 공격 컨셉 5개와 같은 속성 시설이 아니다.

경비실의 역할은 자동 전투에서 직원/방어 몬스터가 개입하는 기반을 만드는 것이다.

```text
평상시: 경비 직원 대기, 순찰 기준점
침입 시: 근처 침입자 교전, 주변 방어 시설과 시너지
파손 시: 경비 합류 지연 또는 비활성화
```

초기에는 경비실이 복잡한 병영 시스템을 가지지 않는다.

간단히 다음 정도만 처리한다.

- 경비 우선순위가 높은 직원이 배정될 수 있다.
- 침입자가 근처에 오면 경비 직원이 자동으로 합류한다.
- 경비실 근처에서 발생한 함정 발동은 전투 결과 요약에 같이 표시된다.

초기 방어 시설은 모두 1성으로 시작하고, 2성 이상은 합성으로 만든다.

예시:

```text
가시 함정 + 독 웅덩이 -> 맹독 가시 함정
가시 함정 + 화염 분사구 -> 달궈진 가시 함정
번개 기둥 + 경비실 -> 경보 코일
냉기 분사구 + 경비실 -> 억류실
경비실 + 훈련장 -> 병영
```

## 방어 시설 발동 방식

방어 시설의 전투 처리는 시설마다 다르게 둔다.

공통 규칙 하나로 `진입 시 발동`, `체류 중 주기 발동`, `쿨타임 발동`, `범위 발동`을 모두 처리하려고 하면 시설 개성이 사라진다.

대신 모든 방어 시설은 같은 발동 데이터 형식을 가지고, 실제 값은 시설별로 다르게 설정한다.

```text
발동 조건: 언제 발동하는가?
대상 규칙: 누구에게 적용하는가?
쿨타임/주기: 얼마나 자주 발동하는가?
효과: 피해, 상태이상, 지연, 교전 합류 등
전투 로그 문구: 플레이어에게 어떻게 보이는가?
```

초기 1성 방어 시설의 발동 프로필:

| 시설 | 발동 조건 | 대상 | 쿨타임/주기 | 핵심 처리 |
| --- | --- | --- | --- | --- |
| 가시 함정 | 침입자가 시설 칸에 진입 | 진입한 침입자 1명 | 진입할 때마다 | 즉발 피해 |
| 독 웅덩이 | 진입 시 1회, 체류 중 주기 | 방 안 침입자 | 짧은 주기 | 피해 + 부식 |
| 화염 분사구 | 쿨타임마다 분사 | 방 안 침입자 전체 | 중간 쿨타임 | 높은 피해 + 연소 |
| 번개 기둥 | 쿨타임마다 방전 | 주 대상 1명 + 근처 대상 | 중간 쿨타임 | 피해 + 축전, 제한적 연계 |
| 냉기 분사구 | 체류 중 주기 | 방 안 침입자 전체 | 짧은 주기 | 낮은 피해 + 감속 |
| 경비실 | 침입자가 근처 방어 구역에 진입 | 배정 직원/근처 침입자 | 반응 지연 | 직원 교전 합류 |

발동 방식 예시:

```text
가시 함정:
침입자가 들어온 순간 바로 찌른다.
가장 단순하고 결과가 바로 보인다.

독 웅덩이:
처음 밟을 때 피해와 부식을 준다.
방 안에 오래 머물면 추가로 피해와 부식 시간이 갱신된다.

화염 분사구:
항상 켜져 있지 않고 일정 주기마다 크게 분사한다.
침입자가 몰려 있을수록 가치가 높다.

번개 기둥:
주기적으로 한 침입자를 때리고 축전을 쌓는다.
근처 침입자나 번개 시설과 제한적으로 연계된다.

냉기 분사구:
방 안에 머무는 침입자에게 계속 감속을 갱신한다.
피해보다 지연이 핵심이다.

경비실:
함정처럼 직접 발동하는 시설이 아니라, 직원이 전투에 합류하는 기준점이다.
```

초기 상태이상 중첩 규칙은 단순하게 둔다.

```text
부식: 중첩하지 않고 지속 시간 갱신
연소: 중첩하지 않고 지속 시간 갱신
감속: 중첩하지 않고 더 강한 감속만 유지
축전: 번개만 누적 스택 사용
```

초기 범위 규칙도 단순하게 둔다.

```text
1셀 시설: 해당 시설 칸 안의 침입자만 대상
넓은 방어 시설: 시설이 차지한 방 영역 안의 침입자만 대상
인접 방/복도까지 타격: 첫 구현에서는 사용하지 않음
```

이렇게 하면 전투 처리는 시설마다 다르게 보이지만, 구현은 같은 인터페이스를 공유할 수 있다.

## 전투 구현 구조

전투 효과 처리는 Visitor 패턴을 핵심 구조로 사용하지 않는다.

방어 시설과 효과는 앞으로 계속 늘어날 가능성이 높다.

```text
피해
부식
연소
축전
감속
공포
흡혈
경비 호출
버프
저항
연계
```

이런 구조에서 Visitor를 핵심으로 쓰면 새 효과가 추가될 때마다 Visitor 인터페이스와 관련 구현을 같이 수정해야 한다.

따라서 첫 구현은 `Strategy + ScriptableObject Effect` 구조로 간다.

핵심 방향:

```text
방어 시설 데이터가 발동 조건, 대상 선택, 효과 목록을 가진다.
전투 런타임은 이벤트를 받아 시설별 발동 조건을 검사한다.
대상 선택기가 침입자/직원/시설 중 적용 대상을 고른다.
효과 ScriptableObject가 실제 피해, 상태이상, 경비 합류 등을 적용한다.
전투 로그는 효과 적용 결과를 받아 기록한다.
```

권장 데이터 구조:

```csharp
DefenseFacilitySO
- triggerProfile
- targetSelector
- effects
- cooldown
- logText
```

```csharp
abstract class CombatEffectSO : ScriptableObject
{
    public abstract void Apply(CombatContext context, IReadOnlyList<CombatTarget> targets);
}
```

예시:

```text
가시 함정
- Trigger: OnEnter
- Target: EnteringIntruder
- Effects: Damage

독 웅덩이
- Trigger: OnEnter + WhileInsideInterval
- Target: IntrudersInRoom
- Effects: Damage + Corrosion

화염 분사구
- Trigger: CooldownPulse
- Target: IntrudersInRoom
- Effects: Damage + Burn

번개 기둥
- Trigger: CooldownPulse
- Target: PrimaryAndNearby
- Effects: Damage + Charge

경비실
- Trigger: IntruderNear
- Target: AssignedGuards
- Effects: JoinCombat
```

### 런타임 상태 분리

ScriptableObject에는 변하지 않는 정의만 둔다.

쿨타임, 축전 누적, 최근 발동 시간, 파손 상태 같은 값은 런타임 인스턴스에 둔다.

```text
DefenseFacilitySO:
설계 데이터, 효과 목록, 기본 수치

DefenseFacilityRuntime:
현재 쿨타임, 현재 축전량, 파손 여부, 전투 중 임시 상태
```

이 분리를 하지 않으면 같은 시설 데이터를 공유하는 여러 방어 시설이 서로 상태를 공유하는 문제가 생길 수 있다.

### 전투 이벤트 흐름

전투 처리는 이벤트 기반으로 굴린다.

```text
침입자 이동/진입/체류
-> CombatEvent 생성
-> 주변 방어 시설 Runtime이 이벤트 확인
-> TriggerProfile이 발동 여부 판단
-> TargetSelector가 대상 선택
-> CombatEffectSO 목록 순서대로 Apply
-> 피해/상태/경비 합류 처리
-> 전투 로그 기록
```

예시:

```text
침입자가 가시 함정 칸에 들어감
-> OnEnter 이벤트
-> 가시 함정 TriggerProfile 발동
-> EnteringIntruder 대상 선택
-> DamageEffect 적용
-> "가시 함정 발동: 침입자 피해" 로그
```

```text
화염 분사구 쿨타임 완료
-> CooldownPulse 이벤트
-> IntrudersInRoom 대상 선택
-> DamageEffect + BurnEffect 적용
-> "화염 분사구 발동: 침입자 피해 + 연소" 로그
```

### Visitor 사용 위치

Visitor는 핵심 전투 적용 로직에는 사용하지 않는다.

다만 나중에 다음 기능에는 사용할 수 있다.

- 효과 툴팁 생성
- 시설 도감 설명 자동 생성
- 밸런스 분석 도구
- 전투 로그 포맷 변환
- 저장 데이터 검증

즉 전투 실행은 효과 객체가 직접 처리하고, Visitor는 효과 데이터를 읽어 다른 표현으로 바꿀 때만 고려한다.

## 초기 공격 컨셉

### 물리/가시

가장 기본적인 직접 피해 계열이다.

역할:

- 즉발 피해
- 튜토리얼용 기본 함정
- 다른 속성 합성의 재료 역할

전투 감각:

```text
밟으면 바로 피해를 준다.
효과가 직관적이고 결과 요약에서 읽기 쉽다.
```

대표 효과:

```text
상태 없음
```

초기에는 출혈 같은 추가 상태를 넣지 않는다. 물리/가시는 단순함이 장점이다.

예시 시설:

```text
1성 가시 함정
2성 쇠가시 바닥
```

합성 방향:

```text
가시 함정 + 독 웅덩이 -> 맹독 가시 함정
가시 함정 + 번개 기둥 -> 전도 가시 함정
가시 함정 + 냉기 분사구 -> 서리 가시 함정
```

### 독/부식

피해를 주면서 침입자의 방어력을 낮추는 약화 계열이다.

역할:

- 기본 피해
- 방어력 감소
- 이후 들어오는 피해 시설과 시너지

전투 감각:

```text
혼자서 큰 피해를 내기보다, 침입자를 약하게 만든다.
부식된 침입자는 뒤이어 맞는 가시, 화염, 번개 피해에 더 취약해진다.
```

대표 효과:

```text
부식: 일정 시간 동안 방어력 감소
```

초기에는 독을 지속 피해로 만들지 않는다.

독/부식 계열의 실제 역할은 방어력 감소다.

예시 시설:

```text
1성 독 웅덩이
2성 맹독 함정
```

합성 방향:

```text
독 웅덩이 + 냉기 분사구 -> 얼어붙은 독 웅덩이
독 웅덩이 + 가시 함정 -> 맹독 가시 함정
독 웅덩이 + 묘지 -> 부패 늪
```

시너지:

- 물리/가시: 부식된 침입자에게 안정적인 추가 피해
- 화염/연소: 부식 후 높은 피해와 지속 피해로 마무리
- 냉기/속박: 느려진 침입자에게 부식 이후 후속 타격 기회 증가
- 뱀파이어/부패 계열 시설: 독 관련 합성 확장

### 화염/연소

짧은 시간에 강한 피해를 주는 공격 계열이다.

역할:

- 높은 피해
- 지속 피해
- 고밸류 피해
- 빠른 정리

전투 감각:

```text
맞는 순간의 피해도 높고, 연소로 추가 피해도 들어간다.
방어력 감소나 감속 이후 들어가면 전투 로그가 크게 움직인다.
```

대표 효과:

```text
연소: 짧은 시간 동안 지속 피해
```

초기에는 화염 확산, 산소, 화재 전파 같은 복잡한 시뮬레이션을 만들지 않는다.

예시 시설:

```text
1성 화염 분사구
2성 불꽃 방
```

합성 방향:

```text
화염 분사구 + 가시 함정 -> 달궈진 가시 함정
화염 분사구 + 기름 저장고 -> 화염 웅덩이
화염 분사구 + 훈련장 -> 투쟁의 화로
```

충돌:

- 슬라임 손님은 화염 시설 근처 동선을 싫어한다.
- 습한 휴식 시설과 같은 구역에 있으면 고객 만족도 충돌이 생길 수 있다.

시너지:

- 오크/전투 시설: 공격적인 던전 분위기와 어울림
- 병목 구간: 여러 침입자를 한 번에 태우기 좋음

### 번개/축전

피해를 주면서 다른 대상이나 장치로 이어지는 연계 계열이다.

초기 5개 공격 컨셉 중 번개만 축전 누적 구조를 가진다.

역할:

- 기본 피해
- 연계 피해
- 축전 스택
- 폭발 피해
- 짧은 기절 또는 행동 지연
- 합성 성장감 제공

전투 감각:

```text
번개가 맞으면 다른 침입자나 번개 시설과 이어질 수 있다.
축전이 쌓이면 방전으로 크게 터진다.
```

대표 효과:

```text
축전: 일정 횟수 또는 수치가 쌓이면 방전 발생
```

초기 방전 효과:

```text
방전: 피해 + 짧은 행동 지연
```

예시 시설:

```text
1성 번개 기둥
2성 축전 코일
```

합성 방향:

```text
번개 기둥 + 가시 함정 -> 전도 가시 함정
번개 기둥 + 경비실 -> 경보 코일
번개 기둥 + 마력 저장소 -> 마력 축전실
```

시너지:

- 냉기/속박: 침입자가 오래 머물러 축전 기회 증가
- 경비 시설: 방전 타이밍에 교전하면 제압력 증가
- 마력 시설: 고등급 번개 시설 합성 재료

주의:

- 축전은 UI 피드백이 중요하다.
- 전투 중 `축전 3/3`, `방전 발생` 같은 짧은 표시가 필요하다.

### 냉기/속박

피해를 주면서 침입자를 느리게 만들고 방어 시간을 버는 지연 계열이다.

역할:

- 기본 피해
- 이동 속도 감소
- 행동 지연
- 다른 피해 시설의 발동 시간을 벌어줌

전투 감각:

```text
피해량은 낮지만 침입자를 오래 붙잡는다.
부식, 축전, 경비 몬스터와 조합할 때 강하다.
```

대표 효과:

```text
감속: 이동 속도 감소
```

초기에는 완전 빙결을 기본 효과로 넣지 않는다. 빙결은 상위 합성 시설에서 제한적으로 사용한다.

예시 시설:

```text
1성 냉기 분사구
2성 서리 바닥
```

합성 방향:

```text
냉기 분사구 + 독 웅덩이 -> 얼어붙은 독 웅덩이
냉기 분사구 + 번개 기둥 -> 정전기 서리장
냉기 분사구 + 경비실 -> 억류실
```

시너지:

- 독/부식: 방어력이 낮아진 침입자를 오래 붙잡아 후속 피해 증가
- 번개/축전: 축전 누적 기회 증가
- 경비/오크 직원: 교전 시간을 늘려 전투 기회 증가

## 확장 공격 컨셉

### 공포/저주

침입자의 전투 능력이나 판단을 약화시키는 디버프 계열이다.

역할:

- 명중률 감소
- 행동 지연
- 사기 저하
- 특정 침입자 타입 대응

대표 상태 후보:

```text
공포: 공격/명중/행동 효율 감소
저주: 받는 피해 증가 또는 회복 방해
```

초기 5개가 안정된 뒤 뱀파이어/데몬 계열 시설과 함께 확장한다.

예시 시설:

```text
공포 묘지
저주 제단
망자의 회랑
```

### 흡혈/피

피해를 주면서 아군을 회복하거나 전투 지속력을 올리는 계열이다.

역할:

- 피해 일부를 회복으로 전환
- 전투형 직원과 시너지
- 장기 교전 강화

대표 상태 후보:

```text
흡혈: 준 피해 일부를 근처 아군 회복으로 전환
출혈: 지속 피해
```

흡혈/피 계열은 오크, 데몬, 전투 식당, 제단 시설과 잘 어울린다.

예시 시설:

```text
피의 제단
흡혈 식당
투쟁의 연회장
```

## 설계 원칙

### 효과는 적게 시작한다

초기 구현에서는 공격 컨셉마다 핵심 효과를 하나만 둔다.

```text
독/부식: 부식
화염/연소: 연소
번개/축전: 축전
냉기/속박: 감속
물리/가시: 상태 없음
```

효과가 많아지면 자동 전투 결과가 읽히기 어려워진다.

처음에는 전투 로그와 결과 요약에서 바로 이해할 수 있는 효과만 사용한다.

별도 누적 구조는 번개/축전만 가진다.

### 공격 컨셉은 합성 갈래다

공격 컨셉은 단순 속성 태그가 아니라 시설 합성의 갈래다.

예시:

```text
가시 함정
-> 맹독 가시 함정
-> 전도 가시 함정
-> 서리 가시 함정
```

같은 기본 시설도 어떤 공격 컨셉을 섞느냐에 따라 역할이 바뀐다.

```text
독을 섞으면 방어력 감소
번개를 섞으면 연계와 축전 폭발
냉기를 섞으면 지연
화염을 섞으면 높은 피해와 지속 피해
```

### 공격 컨셉은 배치를 바꿔야 한다

각 컨셉은 좋은 배치가 달라야 한다.

```text
물리/가시: 침입자가 반드시 밟는 병목
독/부식: 다른 피해 시설보다 앞쪽
화염/연소: 침입자가 몰리는 방
번개/축전: 반복 타격 가능한 구간
냉기/속박: 방어 몬스터가 합류하기 전의 지연 구간
```

이렇게 해야 시설 합성이 단순 수치 상승이 아니라 던전 구조 선택으로 이어진다.

### 고객과 충돌해야 재미있다

공격 시설은 경영과 완전히 분리되지 않는다.

일부 공격 컨셉은 특정 몬스터 고객층과 충돌하거나 시너지를 가져야 한다.

예시:

```text
화염 시설: 슬라임 고객 만족도 감소
냉기 시설: 화염 계열 시설과 구역 충돌 가능
공포 시설: 뱀파이어 고객과 시너지, 일부 손님 만족도 감소
독 시설: 뱀파이어/부패 계열과 시너지
전투 시설: 오크 고객과 시너지
```

이 원칙 덕분에 방어 시설 선택이 경영 선택과 연결된다.

## 전투 피드백

자동 전투에서 공격 컨셉은 짧은 발동 문구로 보여야 한다.

예시:

```text
가시 함정 발동: 침입자 피해
독 웅덩이 발동: 침입자 피해 + 부식
화염 분사구 발동: 침입자 피해 + 연소
축전 코일 발동: 축전 2/3
축전 코일 방전: 침입자 피해 + 행동 지연
냉기 분사구 발동: 침입자 피해 + 감속
```

전투 결과 요약에는 컨셉별 기여도가 드러나야 한다.

예시:

```text
가장 많은 피해: 화염 분사구
가장 오래 지연: 냉기 분사구
가장 많이 적용된 효과: 부식
결정적 발동: 축전 코일 방전
```

## 초기 프로토타입 범위

첫 구현에서는 다음만 만든다.

- 공격 컨셉 5개
- 컨셉별 1성 공격 시설 1개
- 1성 경비실
- 컨셉별 핵심 효과 0-1개
- 시설별 발동 조건, 대상 규칙, 쿨타임/주기
- Visitor가 아닌 Strategy + ScriptableObject Effect 기반 전투 효과 구조
- 전투 중 발동 표시
- 전투 결과 요약에 피해/지연/효과 적용/축전 표시
- 합성 예시 2-3개

초기 시설 예시:

```text
1성 가시 함정: 물리/가시
1성 독 웅덩이: 독/부식
1성 화염 분사구: 화염/연소
1성 번개 기둥: 번개/축전
1성 냉기 분사구: 냉기/속박
1성 경비실: 직원/몬스터 방어 합류
```

초기 합성 예시:

```text
가시 함정 + 독 웅덩이 -> 맹독 가시 함정
번개 기둥 + 경비실 -> 경보 코일
냉기 분사구 + 독 웅덩이 -> 얼어붙은 독 웅덩이
화염 분사구 + 가시 함정 -> 달궈진 가시 함정
```

이 정도면 던전메이커식 시설 성장감과 자동 전투 피드백을 검증할 수 있다.
