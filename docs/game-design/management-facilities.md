# 경영 시설 컨셉

경영 시설은 돈을 버는 건물만 의미하지 않는다.

재고 계열, 창고, 보충 작업은 [재고와 물류](inventory-and-logistics.md) 문서에서 별도로 관리한다.

이 게임에서 경영 시설은 다음 역할을 동시에 가진다.

- 몬스터 손님의 욕구를 처리한다.
- 직원과 방어 인력의 피로와 만족도를 관리한다.
- 재고, 연구, 보충, 수리 같은 작업을 만든다.
- 종족별 선호와 사고 위험을 드러낸다.
- 침입과 반란 때 파손될 수 있는 운영 자산이 된다.
- 시설 합성의 기본 재료이자 성장 출발점이 된다.

즉 경영 시설은 `수익 + 만족도 + 직원 유지 + 합성 기반 + 침입 피해 대상`을 잇는 핵심 시스템이다.

## 설계 원칙

### 시설은 욕구를 처리해야 한다

시설은 단순히 배치 보너스를 주는 장식이 아니라, 캐릭터가 실제로 찾아가서 이용하는 대상이어야 한다.

기본 흐름:

```text
욕구 발생
-> 이용 가능한 시설 탐색
-> 거리, 선호도, 혼잡, 재고, 직원 상태 평가
-> 시설로 이동
-> 대기 또는 이용
-> 돈/재고/만족도/피로/작업 결과 반영
```

이 흐름이 있어야 플레이어가 던전을 관찰할 때 `왜 저 시설이 필요한지`를 이해할 수 있다.

### 경영 시설과 방어 시설은 기본 역할을 분리한다

기본 시설은 경영 시설과 방어 시설을 구분한다.

```text
경영 시설: 손님 욕구, 매출, 직원 유지, 연구, 재고를 담당
방어 시설: 침입자 피해, 지연, 약화, 경비 보조를 담당
```

다만 합성을 통해 상위 시설이 되면 두 역할이 섞일 수 있다.

예시:

```text
고기 식당 + 훈련장 -> 전투 식당
휴식방(습기) + 끈끈이 함정 -> 점액 욕장
휴식방(어둠) + 경비실 -> 공포 휴식실
```

기본은 읽히기 쉽게 분리하고, 성장 과정에서 하이브리드 시설을 만든다.

### 한 시설은 하나의 주 역할을 가져야 한다

초기 시설은 역할이 명확해야 한다.

좋은 예:

```text
저가 음식점: 식사 욕구와 저가 매출
훈련장: 오크 만족도와 전투 준비
연구실: 설계도 분석과 연구 작업
창고: 재고 보관과 보충 작업
```

피해야 할 예:

```text
한 시설이 식사, 연구, 수리, 방어, 휴식을 모두 처리한다.
```

상위 합성 시설은 여러 기능을 가질 수 있지만, 기본 시설은 하나의 기능을 먼저 읽히게 한다.

## 고객 욕구

초기 욕구는 너무 많이 만들지 않는다.

추천 초기 욕구:

```text
식사
구매
휴식
훈련
연구/마력
오락
재고/보충
```

역할:

- 식사: 음식 시설을 이용하고 재고를 소비한다.
- 구매: 상점 시설을 이용하고 매출을 만든다.
- 휴식: 만족도, 체류 시간, 직원 피로를 조절한다.
- 훈련: 전투 성향 종족과 직원 전투 준비를 담당한다.
- 연구/마력: 설계도 분석, 조합식 해금, 마력 시설 선호를 담당한다.
- 오락: 체류 시간, 혼잡 스트레스, 소동 위험을 조절한다.
- 재고/보충: 시설 운영을 위한 물류 작업을 만든다.

## 시설군

### 식당 계열

식사 욕구를 처리하고 재고를 매출로 바꾸는 시설군이다.

주 역할:

- 식사 욕구 처리
- 음식 재고 소비
- 매출 발생
- 종족별 만족도 조절
- 직원/방어 인력의 비번 식사 처리

운영 포인트:

- 재고가 없으면 손님 불만과 직원 불만이 동시에 생긴다.
- 회전율이 낮으면 대기가 늘어난다.
- 종족별 식사 선호가 다르면 식당 구역을 나눌 이유가 생긴다.

초기 시설:

```text
저가 음식점
- 주 고객: 슬라임, 일반 대량 고객
- 효과: 낮은 가격, 높은 회전율, 낮은 만족도 보정
- 재고: 기본 식재료
- 작업: 근무, 보충, 청소
- 위험: 재고 부족 시 불만 빠르게 증가

고기 식당
- 주 고객: 오크
- 효과: 높은 매출, 오크 만족도 증가, 난동 위험 완화
- 재고: 고기 재고
- 작업: 근무, 보충, 청소
- 위험: 재고 부족 시 오크 불만 증가
```

확장 시설:

```text
간식점: 수인과 대량 고객 회전율 중심
피 식당: 뱀파이어/마력 계열 특화
전투 식당: 오크 만족도와 전투 준비를 동시에 제공
```

### 상점 계열

구매 욕구를 처리하고 가장 직접적으로 돈을 버는 시설군이다.

주 역할:

- 구매 욕구 처리
- 물품 재고 소비
- 매출 발생
- 종족별 소비 성향 차이 표현
- 합성 재료가 되는 기본 상업 시설 제공

운영 포인트:

- 상점은 직원 영업 능력의 영향을 크게 받는다.
- 손님이 선호하는 상점이 없으면 체류 시간이 짧아진다.
- 상점 파손은 바로 매출 감소로 이어진다.

초기 시설:

```text
잡화점
- 주 고객: 범용 고객, 수인 확장
- 효과: 낮은-중간 매출, 넓은 고객층
- 재고: 잡화 재고
- 작업: 근무, 보충
- 위험: 혼잡하면 대기 불만 증가

무기점
- 주 고객: 오크
- 효과: 높은 단가, 오크 만족도 증가, 훈련장과 시너지
- 재고: 무기 재고
- 작업: 근무, 보충
- 위험: 침입자가 파괴 대상으로 삼기 좋음
```

확장 시설:

```text
마력 상점: 뱀파이어/데몬과 연구 시설 시너지
전리품 상점: 오펜스 보상을 매출로 전환
희귀품 상점: 고액 소비 고객층 특화
```

### 휴식 계열

만족도와 체류 시간을 조절하는 시설군이다.

주 역할:

- 손님 만족도 회복
- 체류 시간 조절
- 특정 종족 사고 위험 완화
- 직원 비번 행동 제공
- 혼잡과 구역 분리의 이유 제공

운영 포인트:

- 휴식 시설은 직접 매출이 낮거나 없을 수 있다.
- 대신 손님 이탈, 사고, 직원 불만을 줄인다.
- 종족별 휴식 환경은 별도 시설을 늘리기보다 하나의 휴식방에 환경 태그와 운영 정책을 붙여 표현한다.

휴식방 설정:

```text
환경 태그: 기본, 습기, 어둠, 조용함
운영 정책: 공용, 손님 우선, 직원 우선, 특정 종족 우선
```

같은 휴식방이라도 다음처럼 다르게 작동한다.

```text
휴식방 + 습기 태그:
슬라임 만족도 증가, 오염 사고 완화

휴식방 + 어둠 태그:
뱀파이어 만족도 증가, 공포 위험 관리

휴식방 + 직원 우선 정책:
직원 피로 회복 우선, 손님 이용 우선도 낮음
```

초기 시설:

```text
휴식방
- 주 고객: 범용 손님, 직원, 방어 인력
- 효과: 만족도 회복, 체류 시간 조절, 피로 회복
- 설정: 환경 태그와 운영 정책으로 종족/직원 용도를 조절
- 작업: 청소, 관리
- 위험: 설정과 실제 이용층이 맞지 않으면 만족도 회복 효율 저하
```

확장 시설:

```text
점액 욕장: 휴식방(습기)에서 확장, 슬라임 오염을 관리 가능한 보너스로 전환
명상실: 휴식방(어둠/조용함)에서 확장, 연구/마력 계열 직원 피로 회복
치료 휴식실: 휴식방(직원 우선)에서 확장, 부상 회복과 장기 근무 지원
```

### 훈련 계열

전투 성향 종족과 방어 인력의 준비 상태를 관리하는 시설군이다.

주 역할:

- 오크 만족도 증가
- 직원/방어 인력 훈련
- 침입 대응 준비도 상승
- 전투형 합성 시설의 기반 제공

운영 포인트:

- 훈련장은 직접 매출보다 방어와 직원 성장에 의미가 있다.
- 오크 고객층을 키우려면 사실상 필수 시설이 된다.
- 과하게 혼잡하면 근무/훈련 효율이 떨어진다.

초기 시설:

```text
훈련장
- 주 고객: 오크
- 효과: 오크 만족도 증가, 경비/전투 준비도 증가
- 작업: 훈련, 청소
- 위험: 오크가 몰리면 혼잡과 충돌 위험 증가
```

확장 시설:

```text
병영: 경비 대기와 훈련을 결합
전투 훈련장: 침입 중 직원 합류 속도 증가
원정 훈련장: 오펜스 성공률 보정
```

### 연구/마력 계열

설계도 분석, 조합식 해금, 마력 성향 고객을 담당하는 시설군이다.

주 역할:

- 연구 작업 제공
- 설계도 분석
- 시설/조합식 해금
- 뱀파이어와 데몬 만족도 증가
- 후반 합성 트리의 기반 제공

운영 포인트:

- 연구실은 즉시 돈을 벌지 않지만 성장 선택지를 연다.
- 연구 능력, 개인 특성, 종족 보정이 작업 속도에 반영된다.
- 파손되면 해금 진행이 멈춰 장기 성장이 느려진다.

초기 시설:

```text
연구실
- 주 대상: 연구 직원
- 효과: 설계도 분석, 조합식 해금
- 작업: 연구, 청소
- 위험: 연구 우선순위가 낮으면 해금이 지연됨

마력 저장소
- 주 고객: 뱀파이어, 데몬 확장
- 효과: 마력 계열 만족도 증가, 연구/합성 보조
- 작업: 관리, 보충
- 위험: 침입자가 주요 시설로 노릴 수 있음
```

확장 시설:

```text
마력 연구실: 연구 속도와 마력 계열 조합식 보정
설계도 보관실: 분석 가능한 설계도 보관량 증가
저주 연구실: 후반 공포/저주 계열 해금
```

### 오락 계열

체류 시간과 소동 위험을 조절하는 시설군이다.

첫 프로토타입에서는 실제 시설로 넣지 않아도 되지만, 수인과 대량 고객 확장에 필요하다.

주 역할:

- 구매 외 행동 제공
- 체류 시간 증가
- 혼잡 스트레스 완화
- 수인 소동 위험 완화

확장 시설:

```text
놀이방
- 주 고객: 수인, 대량 고객
- 효과: 체류 시간 증가, 소동 위험 완화
- 작업: 관리, 청소

도박장
- 주 고객: 위험 선호 고객층
- 효과: 높은 변동성 매출
- 위험: 사고와 불만 변동성 증가
```

### 창고/보충 계열

재고와 직원 작업을 연결하는 시설군이다.

주 역할:

- 재고 보관량 증가
- 보충 작업 생성
- 시설별 재고 부족 완화
- 수인/힘 높은 직원의 작업 차이 표현
- 침입 피해 이후 복구 비용 관리

운영 포인트:

- 창고 위치는 직원 동선에 직접 영향을 준다.
- 창고가 멀면 상점과 식당이 재고 부족 상태에 빠지기 쉽다.
- 힘과 이동속도 능력의 차이를 체감시키는 핵심 시설군이다.

초기 시설:

```text
창고
- 주 대상: 직원
- 효과: 재고 보관량 증가, 보충 작업의 출발점
- 작업: 보충, 정리
- 위험: 침입자가 파괴하면 여러 시설 운영이 동시에 흔들림
```

확장 시설:

```text
식재료 창고: 식당 재고 특화
무기 창고: 무기점/경비 시설 특화
마력 보관고: 연구/마력 시설 특화
```

### 유지보수 계열

수리, 청소, 사고 복구를 보조하는 시설군이다.

첫 구현에서는 별도 유지보수 시설 없이 직원 작업만으로 처리해도 된다.

하지만 중반 이후에는 유지보수 시설이 있어야 큰 던전의 운영 부담을 조절할 수 있다.

확장 시설:

```text
수리 작업대
- 효과: 수리 속도 증가, 수리 비용 감소
- 작업: 수리

청소 도구함
- 효과: 청소 작업 속도 증가, 오염 사고 복구 보조
- 작업: 청소

응급실
- 효과: 부상 회복, 직원 이탈 위험 완화
- 작업: 구조/응급
```

## 초기 시설 목록

첫 프로토타입에서는 경영 시설을 너무 많이 만들지 않는다.

추천 초기 시설:

```text
저가 음식점
고기 식당
잡화점
무기점
휴식방
훈련장
연구실
마력 저장소
창고
```

이 9개면 다음을 검증할 수 있다.

- 슬라임: 저가 음식점, 휴식방(습기), 청소/오염
- 오크: 고기 식당, 무기점, 훈련장, 난동
- 뱀파이어: 휴식방(어둠), 연구실, 마력 저장소, 공포
- 직원 운영: 휴식방(직원 우선), 창고, 보충, 피로
- 성장: 연구실, 마력 저장소, 합성 해금

수인과 데몬은 첫 프로토타입에서 실제 방문 대상으로 넣지 않더라도, `잡화점`, `마력 저장소`, `놀이방`, `희귀품 상점` 같은 확장 방향을 미리 고려한다.

## 종족별 연결

초기 3종 연결:

```text
슬라임
- 선호: 저가 음식점, 휴식방(습기)
- 기피: 화염 시설 근처 동선
- 운영 문제: 오염, 낮은 객단가, 대량 방문

오크
- 선호: 고기 식당, 무기점, 훈련장
- 기피: 조용한 전용 구역, 낮은 식사 공급
- 운영 문제: 난동, 높은 재고 소비, 혼잡

뱀파이어
- 선호: 휴식방(어둠), 연구실, 마력 저장소
- 기피: 밝은 시설, 과도한 혼잡
- 운영 문제: 공포, 구역 분리, 낮은 식사 부담 대신 전용 시설 요구
```

확장 2종 연결:

```text
수인
- 선호: 잡화점, 간식점, 놀이방
- 강점: 빠른 보충, 정찰, 상점 운영
- 운영 문제: 혼잡 시 소동

데몬
- 선호: 마력 저장소, 희귀품 상점, 제단 계열
- 강점: 고액 소비, 전투/연구
- 운영 문제: 저주, 전용 구역 요구, 높은 사고 리스크
```

## 직원 작업 연결

경영 시설은 직원 작업을 만든다.

작업 연결:

```text
식당: 근무, 보충, 청소
상점: 근무, 보충
휴식 시설: 청소, 관리
훈련장: 훈련, 청소
연구실: 연구, 청소
마력 저장소: 관리, 보충
창고: 보충, 정리
수리 작업대: 수리
청소 도구함: 청소
```

이 연결 덕분에 직원 능력이 시설 운영에 바로 반영된다.

예시:

```text
영업 높은 직원 -> 상점/식당 매출 증가
이동속도 높은 직원 -> 보충 지연 감소
힘 높은 직원 -> 한 번에 운반하는 재고량 증가
연구 높은 직원 -> 설계도 분석 속도 증가
청소 높은 직원 -> 오염 사고 복구 속도 증가
수리 높은 직원 -> 파손 시설 복구 속도 증가
```

## 합성과의 연결

경영 시설은 합성의 출발점이다.

기본 방향:

```text
기본 경영 시설
-> 종족 특화 경영 시설
-> 경영 + 방어 하이브리드 시설
-> 원정/후반 보상 시설과 결합한 상위 시설
```

예시:

```text
저가 음식점 + 휴식방(습기) -> 점액 식당
고기 식당 + 훈련장 -> 전투 식당
무기점 + 훈련장 -> 전투 보급소
휴식방(어둠) + 마력 저장소 -> 마력 휴식실
연구실 + 마력 저장소 -> 마력 연구실
창고 + 저가 음식점 -> 대형 식당
창고 + 무기점 -> 무기 보급소
```

합성 결과는 단순히 더 좋은 시설이 아니라, 다른 운영 전략을 열어야 한다.

좋은 합성 결과:

- 특정 종족 고객층이 강해진다.
- 특정 직원 작업이 쉬워진다.
- 특정 침입 대응 방식과 연결된다.
- 다른 합성 트리로 갈라진다.

## 배치 의미

경영 시설은 배치가 중요해야 한다.

배치 판단:

```text
입구 근처: 대량 고객 회전율 증가, 침입 피해 위험 증가
중앙 구역: 접근성 좋음, 혼잡 위험 증가
깊은 구역: 특정 종족 전용 운영에 좋음, 일반 손님 접근성 낮음
창고 근처: 보충 효율 증가
휴식방(직원 우선) 근처: 근무 복귀 빠름
방어 구역 근처: 침입 시 피해 위험 증가, 하이브리드 시설 효과 활용 가능
```

좋은 던전은 단순히 시설 수가 많은 던전이 아니라, 손님 동선, 직원 동선, 침입자 동선을 함께 고려한 던전이다.

## 첫 구현 범위

첫 구현 목표:

- 고객이 시설을 선택해 이동한다.
- 시설에 수용 인원과 대기열이 있다.
- 식당/상점은 재고를 소비하고 매출을 만든다.
- 휴식방은 환경 태그와 운영 정책에 따라 손님 만족도와 직원 피로에 영향을 준다.
- 연구실은 설계도 분석 작업을 만든다.
- 창고는 보충 작업의 기준점이 된다.
- 종족별 선호 시설이 만족도에 반영된다.
- 파손 시설은 이용 불가 상태가 된다.

첫 구현에서 생략해도 되는 것:

- 유지비 세부 계산
- 시설별 복잡한 직원 스킬 요구치
- 유지보수 전용 시설
- 오락 시설 전체
- 시설 내구도 단계
- 3성 이상 경영 시설
- 수인/데몬 전용 경영 루프
