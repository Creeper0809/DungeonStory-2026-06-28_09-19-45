# 침입 이벤트

## 침입 이벤트 발생 방식

침입 이벤트는 고정 날짜/웨이브가 아니라 `내부 위협도 누적 방식`으로 발생한다.

림월드처럼 난이도에 따라 던전 가치와 시간의 반영 정도가 달라지는 구조를 참고한다. 고정된 날짜마다 같은 강도의 침입이 오는 방식보다는, 던전의 성장 상태와 플레이 시간이 함께 반영되는 구조가 적합하다.

위협도는 시스템 내부 값이며, 플레이어에게 실제 게이지로 그대로 보여주지 않는다. 플레이어는 소문, 정찰 보고, 경고 메시지, 분위기 변화 같은 간접 신호로 침입 가능성을 파악한다.

가능한 변수:

- 던전 가치: 보물, 시설, 총 수익, 방 개수
- 던전 평판: 손님 수, 만족도, 유명도
- 시간: 마지막 침입 이후 경과 시간
- 난이도: 던전 가치와 시간의 반영 비율
- 위험도: 최근 방어 실패, 방어 시설 부족, 침입 경고 누적

초기 설계안:

```text
평상시 운영
-> 난이도에 따라 위협도 상승
-> 경고 단계
-> 위협도 임계치 도달
-> 짧은 랜덤 딜레이 후 침입 발생
-> 전투/방어 결과 처리
-> 위협도 초기화
-> 짧은 안전 시간
-> 다시 평상시 운영
```

침입 주기 방향:

- 초반에는 드물고 약하게 발생한다.
- 던전 가치가 높아질수록 침입 빈도와 강도가 올라간다.
- 시간이 지날수록 기본 위협도도 서서히 오른다.
- 난이도가 높을수록 던전 가치와 시간의 영향이 강해진다.
- 플레이어가 대비할 수 있도록 침입 전 경고를 주되, 정확한 수치 게이지는 보여주지 않는다.
- 위협도가 높은 상태에서도 정확히 같은 순간에 침입하지 않도록 짧은 랜덤 딜레이를 둔다.
- 침입 직후에는 짧은 안전 시간을 둔다.
- 주기가 너무 고정되면 숙제처럼 느껴질 수 있으므로 약간의 변동폭을 둔다.

위협도 단계 예시:

```text
내부 위협도 0-69%: 평상시
내부 위협도 70-99%: 침입 경고 가능
내부 위협도 100% 이상: 침입 후보 상태
100% 이후 랜덤 딜레이: 실제 침입 발생
```

플레이어에게 보이는 경고 예시:

```text
모험가들의 소문이 늘고 있습니다.
수상한 정찰대가 던전 근처에서 목격되었습니다.
침입이 임박한 것 같습니다.
```

난이도 예시:

```text
쉬움:
던전 가치 반영 낮음
시간 반영 낮음
침입 간격 김
침입 전 경고 넉넉함

보통:
던전 가치 반영 보통
시간 반영 보통
침입 간격 보통
침입 전 경고 기본

어려움:
던전 가치 반영 높음
시간 반영 높음
침입 간격 짧음
침입자 유형 다양화 빠름
```

침입 강도 계산 방향:

```text
위협도 상승량 = 기본 상승량 + 던전 가치 보정 + 시간 보정 + 난이도 보정 + 랜덤 변동폭
침입 강도 = 현재 위협도 + 던전 가치 보정 + 시간 보정 + 난이도 보정 + 랜덤 변동폭
```

던전 가치와 시간은 함께 반영하되, 난이도가 두 값의 비중을 조절한다. 이렇게 하면 잘 성장한 던전은 더 큰 위협을 부르고, 느리게 플레이하더라도 시간이 지나며 긴장감이 유지된다.
