# 모듈형 시설물 전체 목록

## 목표

현재 `연구실`, `식당`, `훈련장`, `창고`처럼 방 전체를 한 번에 설치하는 통짜 건물을 없애고, 플레이어가 벽과 문으로 방을 만든 뒤 작은 시설물을 직접 배치하도록 전환한다.

방 이름과 성향은 완제품 건물 이름이 아니라 다음 입력을 합산해 결정한다.

- 핵심 설비가 제공하는 행동과 작업
- 테이블, 좌석, 보관함 같은 수용량
- 연구, 마력, 전투, 위생, 물류 등의 성향 기여
- 미관, 고급스러움, 혼잡도, 청결도
- 실제 이용 기록과 운영 사건

## 제작 원칙

- **핵심 설비**만 직원 작업, 손님 이용, 재고 소비 같은 런타임 행동을 소유한다.
- **보조 가구**는 방 성향, 수용량, 환경 수치만 기여한다. 의자 하나가 식당 전체처럼 동작하지 않는다.
- 모든 신규 시설은 `selfContainedRoom = false`로 둔다. 닫힌 벽과 내벽 문이 있는 방에서만 방 효과를 받는다.
- 한 그리드 칸은 16px를 기준으로 한다. 점유 폭은 1~3칸이며, 세로 그림은 위로 뻗어도 바닥 점유 칸은 늘리지 않는다.
- 스프라이트는 투명 배경, Point 필터, PPU 16, 하단 중앙 피벗을 기본으로 한다.
- 설치 슬롯은 `바닥`, `벽걸이`, `천장`, `바닥 오버레이`로 분리한다. 장식이 바닥 가구의 설치 칸을 빼앗지 않게 한다.
- 아래의 `1차`는 현재 통짜 기본 시설을 해체하는 데 필요한 항목, `2차`는 상위 계보와 고급화, `3차`는 분위기와 특수 계보용이다.

## 유지할 구조물

이 항목은 방의 내용물이 아니므로 통짜 시설 분해 대상에서 제외한다.

| 항목 | 처리 |
|---|---|
| 외벽 | 지금처럼 건물 외곽에서 자동 생성 |
| 복도/바닥 | 이동 가능 영역과 층 바닥 담당 |
| 내벽 | 1칸 구조 경계로 유지 |
| 내벽 문 | 내벽 한 칸에 설치하는 1칸 문으로 유지 |
| 던전 출입문 | 기존 3x3 출입구로 별도 유지 |
| 계단/엘리베이터 | 층간 이동 설비로 별도 유지 |

가시, 독, 화염, 냉기, 번개, 경보 코일 같은 독립 함정도 이미 단일 기능 설비이므로 그대로 유지한다. `경비실`, `병영`처럼 방 전체인 방어 시설만 아래 파츠로 분해한다.

## 마이그레이션 규칙

1. 통짜 건물은 건축 목록에서 숨기고 `방 레시피/계보 이름`으로만 남긴다.
2. 신규 게임은 작은 시설물만 직접 설치한다.
3. 상위 시설 해금은 완제품 배치 권한이 아니라 고급 파츠, 외형 변형, 운영 계보를 해금한다.
4. 기존 `BuildingSO.Evolution`의 태그·점수·메트릭을 각 파츠로 분산한다.
5. 룸 역할은 핵심 설비가 만들고, 보조 가구는 복합 성향과 품질을 보탠다.
6. 기존 씬의 통짜 시설은 자동 변환하지 않는다. 테스트 씬은 분해표의 기본 구성으로 다시 배치해 겹침과 동선을 직접 검증한다.
7. 저장 데이터 호환이 필요해지면 통짜 건물을 동일 위치에 억지로 잘게 생성하지 않고, 환불 후 재배치 또는 방 단위 마이그레이션 UI를 사용한다.

## 구현 순서

1. 설치 슬롯과 겹침 규칙: 바닥, 벽걸이, 천장, 바닥 오버레이.
2. 1차 파츠의 스프라이트와 `BuildingSO` 제작.
3. 핵심 설비와 보조 가구의 런타임 책임 분리.
4. 방 성향·수용량·환경 점수 집계 연결.
5. 기본 통짜 건물을 건축 목록에서 제거하고 테스트 씬을 모듈형 방으로 교체.
6. 2차 고급 파츠와 상위 시설 계보 연결.
7. 3차 특수 장식과 사건 기반 계보 연결.
