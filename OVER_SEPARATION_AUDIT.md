# Over-Separation Audit

## Scope
- Target: all project-owned non-Editor C# scripts under `Assets/Scripts/**/*.cs` plus `Assets/DataManager.cs`.
- Excluded: `Assets/**/Editor/**` and vendor/plugin scripts. Vendor function coverage remains in `SCRIPT_FUNCTION_AUDIT_VENDOR.md`.
- Scan count: 299 files, 165 interfaces, 522 classes, 108 structs, 64 enums.
- This report does not duplicate the full function list. It audits interfaces, providers, factories, services, runtime wrappers, and thin layers on top of `SCRIPT_FUNCTION_AUDIT.md`.

## Method
- Extracted interface/class/struct/enum declarations and implementation relationships from every scoped file.
- Treated `Service`, `Provider`, `Factory`, `Builder`, `Formatter`, `Resolver`, `Query`, `Reader`, `Source`, `Adapter`, `Catalog`, `Lookup`, and `Validator` suffixes as the main over-separation search surface.
- Did not classify a type as mergeable only because it has one implementation. Branching, null-safe aggregation, DTO construction, scene lookup failure policy, Unity object creation, and lifecycle/serialization boundaries move a type to keep or watch.
- `merge-candidate` means a layer can likely be folded without behavior change. `watch-*` means the layer is acceptable today but should be challenged before adding more similar abstractions.

## Main Findings
- Real over-separation exists, but it is narrower than the first broad scan suggested.
- The clearest merge set is 3 pure factory-alias services, 1 UI touch guard wrapper, and 6 `*SummaryRuntimeSource` layers.
- The `*SummaryService` classes themselves should stay: they build null-safe DTO snapshots and keep UI tab text code away from runtime objects.
- The `*RuntimeProvider` family is small and repetitive. Do not delete it wholesale; reduce boilerplate with a shared helper/base if new providers continue to appear.
- Factories, catalogs, validators, builders, panel/presenter factories, and Unity adapters mostly remain valid DI boundaries because they hide object creation, VContainer injection, resource loading, or failure policy.

## Cluster Notes
### Summary Runtime Sources
- `BuildingManagementSummaryRuntimeSource`, `InvasionDefenseSummaryRuntimeSource`, `OffenseTabSummaryRuntimeSource`, `OperationTabSummaryRuntimeSource`, `ResearchCraftingSummaryRuntimeSource`, and `CodexRecordSummaryRuntimeSource` are consumed only by their sibling services.
- Recommendation: remove the source interface/class layer and let each `*SummaryService` inject `IDungeonSceneComponentQuery` or `DungeonSceneRuntimeReferences` directly. Keep the summary service and DTOs.

### Factory Alias Services
- `CharacterFeedbackBubbleService`, `CharacterSocialMemoryService`, and `FacilityEvolutionStateService` expose the same `GetOrAdd` operation as their factories.
- Recommendation: inject the factory interfaces at call sites, or move the domain alias onto the factory interface if the domain wording is important.

## Recommended Refactor Order
1. Fold `*SummaryRuntimeSource` layers into sibling `*SummaryService` classes. This removes six interfaces/classes plus six VContainer registrations while keeping the useful summary boundary.
2. Replace pure alias services (`CharacterFeedbackBubbleService`, `CharacterSocialMemoryService`, `FacilityEvolutionStateService`) with their factories at call sites, or move the domain alias onto the factory interface.
3. Fold `IUiTouchGuardService` into `IUiPopupService` or one explicit UI interaction gate if touch blocking remains popup/UI related.
4. Inspect the runtime provider cluster after those smaller folds. Prefer a shared cached scene-runtime provider helper before deleting named contracts that encode domain language.
5. Keep `*SummaryService`, builders, validators, factories, catalogs, and Unity adapters unless their consumers prove they no longer isolate a volatile dependency.

## Decision
- The exhaustive pass found both valid boundaries and real over-separation. The final strict result was 10 merge-candidate classes, 10 merge-candidate interfaces, many watch items, and many keep boundaries.
- Phase 7 implemented all merge-candidate refactors from this report. Watch and keep-boundary items remain intentionally unchanged.
